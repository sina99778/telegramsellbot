# TelegramSellBot

English README: [README.md](README.md)

TelegramSellBot یک ربات فروش و مدیریت اشتراک VPN/Proxy برای تلگرام است. پروژه فقط یک ربات ساده نیست؛ بات تلگرام، API، Mini App، وبهوک پرداخت، پردازش های پس زمینه، اتصال به Sanaei X-UI، فروش کانفیگ آماده، کیف پول، تیکتینگ، گزارش گیری و ابزارهای مدیریت را در یک مخزن کنار هم می آورد.

## قابلیت های اصلی

- فروش پلن و تحویل خودکار کانفیگ از X-UI بعد از پرداخت موفق
- فروش کانفیگ آماده از فایل متنی، بدون نیاز به اتصال پنل X-UI
- خرید حجم و زمان دلخواه با قیمت گذاری جداگانه توسط مدیر
- تمدید سرویس، خرید حجم اضافه و خرید زمان اضافه
- کیف پول داخلی، شارژ کیف پول و پرداخت از کیف پول
- پرداخت با NOWPayments، TetraPay، پرداخت دستی کریپتو و کارت به کارت
- تایید دستی رسید پرداخت برای پرداخت های دستی و کارت به کارت
- بازبینی مجدد پرداخت های درگاهی توسط مدیر و کاربر
- تایید شماره موبایل کاربر قبل از خرید، با حالت فقط ایران یا هر شماره ای
- کد تخفیف، محدودیت فروش پلن، موجودی پلن و نمایش موجودی به کاربر
- هدیه حجم و زمان به کانفیگ های فعال، همه سرورها یا یک سرور خاص
- کانفیگ تست با امکان فعال/غیرفعال کردن و ریست محدودیت کاربر
- تیکت پشتیبانی در ربات و Mini App برای کاربر و مدیر
- پنل کاربری و پنل مدیریت Mini App
- گزارش خرید هفتگی Excel همراه با نام کانفیگ خریداری شده
- اعلان پایان حجم/زمان و دکمه تمدید
- بکاپ، broadcast، retargeting، recovery و reconciliation

## نصب سریع روی سرور

روی یک سرور Ubuntu تازه و با کاربر `root` اجرا کنید:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/sina99778/telegramsellbot/master/setup.sh)
```

این نصب کننده:

- پیش نیازهای اصلی را نصب می کند
- پروژه را داخل `/opt/telegramsellbot` قرار می دهد
- فایل `.env` را با سوال های تعاملی می سازد
- Docker، Nginx و Certbot را آماده می کند
- سرویس های `api`، `bot`، `worker`، `postgres` و `redis` را بالا می آورد

## نصب دستی

```bash
sudo -i
apt-get update
apt-get install -y git curl rsync
git clone https://github.com/sina99778/telegramsellbot.git /opt/telegramsellbot
cd /opt/telegramsellbot
chmod +x setup.sh install.sh deploy.sh
bash install.sh
```

بعد از نصب، برای deploy کامل:

```bash
cd /opt/telegramsellbot
./deploy.sh full
```

برای restart سریع بدون rebuild کامل:

```bash
cd /opt/telegramsellbot
./deploy.sh reload
```

برای گرفتن آخرین آپدیت ها:

```bash
cd /opt/telegramsellbot
git pull
./deploy.sh full
```

## تنظیمات `.env`

فایل `.env` روی سرور ساخته می شود. مهم ترین گزینه ها:

- `BOT_TOKEN`: توکن ربات از BotFather
- `BOT_USERNAME`: یوزرنیم ربات بدون `@`
- `OWNER_TELEGRAM_ID`: آیدی عددی مالک
- `ADMIN_API_KEY`: کلید داخلی برای API مدیریت
- `DATABASE_URL`: اتصال PostgreSQL داخل Docker
- `REDIS_URL`: اتصال Redis داخل Docker
- `XUI_BASE_URL`, `XUI_USERNAME`, `XUI_PASSWORD`: اتصال به پنل X-UI
- `NOWPAYMENTS_API_KEY`, `NOWPAYMENTS_IPN_SECRET`, `NOWPAYMENTS_IPN_CALLBACK_URL`: تنظیمات NOWPayments
- `TETRAPAY_API_KEY`, `TETRAPAY_CALLBACK_URL`: تنظیمات TetraPay
- `WEB_BASE_URL`: آدرس دامنه پنل و Mini App
- `SUPPORT_URL`: آدرس پشتیبانی یا پنل

نکته: تنظیمات کارت به کارت، پرداخت دستی کریپتو، تایید شماره، خرید دلخواه، کانفیگ تست، رفرال و نرخ تومان از داخل پنل مدیریت ربات انجام می شوند و نیازی به تغییر `.env` ندارند.

## راه اندازی اولیه ربات

1. از BotFather یک ربات بسازید و توکن را در `.env` بگذارید.
2. دامنه را به سرور وصل کنید و `WEB_BASE_URL` را با همان دامنه تنظیم کنید.
3. اگر از درگاه ها استفاده می کنید، callbackها را روی دامنه خودتان بگذارید:
   `https://your-domain.com/api/webhooks/nowpayments`
   `https://your-domain.com/api/webhooks/tetrapay`
4. `./deploy.sh full` را اجرا کنید.
5. داخل تلگرام `/start` بزنید. اگر `OWNER_TELEGRAM_ID` درست باشد، مالک به منوی مدیریت دسترسی دارد.
6. از پنل مدیریت سرورها، پلن ها، درگاه ها و تنظیمات فروش را آماده کنید.

- مدیریت پلن ها: ساخت پلن، تغییر نام، قیمت، مدت، حجم، موجودی و وضعیت فعال
- فروش کانفیگ آماده: ساخت پلن آماده و آپلود فایل کانفیگ
- مدیریت کاربران: جستجو، مشاهده وضعیت، تغییر موجودی، ارسال پیام و مدیریت کاربر
- مشتریان و سرویس ها: مشاهده خریدها، سرویس های فعال و جزئیات کانفیگ
- مالی و پرداخت ها: مشاهده پرداخت ها، بازبینی درگاه، تایید یا رد پرداخت دستی
- تنظیمات ربات: قیمت تمدید، نرخ تومان، خرید دلخواه، درگاه ها، تایید شماره، کانفیگ تست، رفرال و جوین اجباری
- تیکت ها: پاسخ به تیکت و بستن تیکت
- تخفیف ها: ساخت و مدیریت کد تخفیف
- هدیه حجم و زمان: ارسال هدیه به همه کانفیگ ها، فقط فعال ها، همه سرورها یا یک سرور خاص
- Recovery و Reconciliation: بازیابی تحویل ها و بررسی مجدد پرداخت ها
- گزارش ها و بکاپ: گزارش Excel، آمار و بکاپ دستی

## تنظیم درگاه های پرداخت

از مسیر `پنل مدیریت -> تنظیمات ربات -> مدیریت درگاه های پرداخت` هر درگاه منوی جدا دارد.

### NOWPayments

1. وارد منوی NOWPayments شوید.
2. فعال یا غیرفعال بودن را تنظیم کنید.
3. در صورت نیاز API Key و IPN Secret اختصاصی را وارد کنید. اگر خالی باشد از مقدار `.env` استفاده می شود.
4. در پنل NOWPayments آدرس IPN را روی `NOWPAYMENTS_IPN_CALLBACK_URL` تنظیم کنید.
5. بعد از پرداخت موفق، وبهوک پرداخت را تایید می کند و کانفیگ یا شارژ کیف پول خودکار انجام می شود.

### TetraPay

1. وارد منوی TetraPay شوید.
2. فعال یا غیرفعال بودن را تنظیم کنید.
3. API Key را وارد کنید یا از مقدار `.env` استفاده کنید.
4. نرخ دلار به تومان را از تنظیمات ربات تنظیم کنید.
5. کاربر درگاه ریالی را انتخاب می کند و بعد از تایید پرداخت، تحویل خودکار انجام می شود.

### پرداخت دستی کریپتو

1. وارد منوی پرداخت دستی کریپتو شوید.
2. درگاه را فعال کنید.
3. ارز را انتخاب کنید، مثلا `USDT TRC20`.
4. آدرس ولت را وارد کنید. چند ولت مختلف قابل ثبت است.
5. کاربر ارز مورد نظر را انتخاب می کند، مبلغ و آدرس را می بیند و TX Hash را ارسال می کند.
6. مدیر درخواست را تایید یا رد می کند.
7. اگر پرداخت برای شارژ کیف پول باشد، کیف پول شارژ می شود. اگر برای خرید مستقیم باشد، بعد از تایید کانفیگ ساخته و ارسال می شود.

### کارت به کارت

1. وارد `مدیریت درگاه های پرداخت -> کارت به کارت` شوید.
2. شماره کارت، نام صاحب کارت، نام بانک و توضیح پرداخت را وارد کنید.
3. کارت به کارت را فعال کنید.
4. کاربر هنگام خرید یا شارژ کیف پول، گزینه کارت به کارت را انتخاب می کند.
5. ربات مبلغ تومان، شماره کارت و مشخصات کارت را نمایش می دهد.
6. کاربر بعد از پرداخت عکس رسید را ارسال می کند.
7. رسید برای مدیرها ارسال می شود.
8. مدیر روی `تایید` یا `رد` می زند.
9. بعد از تایید، برای خرید مستقیم کانفیگ خودکار ساخته و ارسال می شود؛ برای شارژ، کیف پول کاربر شارژ می شود.

## تایید شماره موبایل

از مسیر `پنل مدیریت -> تنظیمات ربات -> تایید شماره موبایل`:

- پیش فرض این قابلیت غیرفعال است.
- مدیر می تواند آن را فعال یا غیرفعال کند.
- حالت `فقط شماره ایران` فقط شماره های موبایل ایران را قبول می کند.
- حالت `هر شماره ای` شماره های بین المللی یا عددی را می پذیرد.

وقتی فعال باشد، کاربر قبل از دیدن لیست پلن ها باید شماره موبایل خودش را ارسال کند. شماره تایید شده در پروفایل کاربر ذخیره می شود و دفعات بعد دوباره درخواست نمی شود.

## ساخت پلن عادی

1. وارد مدیریت پلن ها شوید.
2. پلن جدید بسازید.
3. inbound مربوط به سرور X-UI را انتخاب کنید.
4. نام، مدت، حجم و قیمت را وارد کنید.
5. اگر می خواهید تعداد فروش محدود باشد، موجودی/Stock را تنظیم کنید.
6. مقدار موجودی `0` یعنی نامحدود و در ربات به کاربر محدودیت نمایش داده نمی شود.
7. پلن فعال در فروشگاه ربات و Mini App نمایش داده می شود.

## تغییر قیمت و مدت پلن

از جزئیات پلن می توانید:

- مدت پلن را تغییر دهید
- حجم پلن را تغییر دهید
- قیمت خرید را تغییر دهید
- قیمت تمدید یا قیمت خریدهای دلخواه را در تنظیمات جدا کنید
- موجودی فروش را محدود یا نامحدود کنید

تغییرات روی خریدهای بعدی اثر دارد و اشتراک های قبلی را تغییر نمی دهد.

## فروش کانفیگ آماده

از این حالت وقتی استفاده کنید که نمی خواهید یا نمی توانید X-UI را وصل کنید، یا از قبل لیست کانفیگ آماده دارید.

1. وارد `فروش کانفیگ آماده` شوید.
2. یک پلن آماده بسازید و نام، مدت، حجم و قیمت بدهید.
3. یک فایل `.txt` آپلود کنید که هر خط آن یک کانفیگ کامل باشد.
4. Pool آن پلن را فعال نگه دارید.
5. کاربر پلن را مثل پلن عادی می خرد.
6. بعد از پرداخت، ربات اولین کانفیگ آزاد را برمی دارد، آن را فروخته شده علامت می زند و برای کاربر می فرستد.

نکات مهم:

- هر خط فقط یک بار فروخته می شود.
- خطوط خالی و کانفیگ های تکراری هنگام آپلود نادیده گرفته می شوند.
- وقتی موجودی فایل تمام شود، پلن برای کاربر قابل خرید نیست.
- اگر برای پلن محدودیت فروش گذاشته باشید، موجودی موثر بین محدودیت پلن و تعداد کانفیگ های آماده محاسبه می شود.

## خرید حجم و زمان دلخواه

از `تنظیمات ربات -> خرید دلخواه`:

1. قابلیت را فعال کنید.
2. قیمت هر 1 گیگ را تعیین کنید.
3. قیمت هر 1 روز را تعیین کنید.
4. کاربر در فروشگاه گزینه خرید حجم و زمان دلخواه را می بیند.
5. کاربر حجم، مدت، نام کانفیگ و روش پرداخت را انتخاب می کند.
6. بعد از پرداخت، کانفیگ با همان حجم و مدت ساخته می شود.

برای این قابلیت باید حداقل یک پلن فعال متصل به inbound وجود داشته باشد تا ربات از آن به عنوان الگوی ساخت کانفیگ استفاده کند.

## کانفیگ تست

از `تنظیمات ربات -> فعال/غیرفعال کانفیگ تست`:

- مدیر می تواند کانفیگ تست را فعال یا غیرفعال کند.
- هر کاربر به صورت پیش فرض فقط یک بار می تواند تست بگیرد.
- مدیر می تواند محدودیت تست کاربر را ریست کند.
- اگر تست غیرفعال باشد، گزینه مربوطه برای کاربر کار نمی کند یا پیام غیرفعال بودن نمایش داده می شود.

## تمدید و هدیه حجم/زمان

کاربر از بخش سرویس های من یا از اعلان پایان حجم/زمان می تواند تمدید کند.

مدیر می تواند:

- به کانفیگ های فعال هدیه بدهد
- به همه کانفیگ ها هدیه بدهد
- هدیه را به همه سرورها اعمال کند
- هدیه را فقط روی یک سرور خاص اعمال کند
- حجم، زمان یا هر دو را هدیه بدهد

## کیف پول

کاربر می تواند کیف پول را با درگاه ها، پرداخت دستی کریپتو یا کارت به کارت شارژ کند. سپس در خرید پلن، گزینه کیف پول را انتخاب می کند.

کاربردهای کیف پول:

- خرید سریع بدون رفتن به درگاه
- شارژ دستی توسط مدیر
- بازگشت وجه خودکار در صورت خطای ساخت کانفیگ
- پاداش رفرال

## کد تخفیف

مدیر می تواند کد تخفیف بسازد و تنظیم کند:

- درصد تخفیف
- تعداد استفاده
- محدودیت زمانی
- فعال یا غیرفعال بودن

کاربر هنگام خرید می تواند کد تخفیف را وارد کند یا بدون کد ادامه دهد.

## تیکت پشتیبانی

## بکاپ و بازیابی (Backup & Restore)

برای اطمینان از حفظ اطلاعات کاربران و کانفیگ‌ها، سیستم بکاپ و بازیابی به پروژه اضافه شده است.

### تهیه بکاپ (Backup)

برای بکاپ‌گیری از دیتابیس و فایل `.env` (تنظیمات)، دستور زیر را در سرور اجرا کنید:

```bash
cd /opt/telegramsellbot
./backup.sh
```

این اسکریپت یک فایل زیپ از دیتابیس (پسوند `.sql.gz`) و یک کپی از فایل `.env` در پوشه `backups/` ایجاد می‌کند. این فایل‌ها را به جای امنی (مثلاً کامپیوتر شخصی خود) منتقل کنید.

### بازیابی بکاپ (Restore) در سرور جدید

اگر سرور را تغییر دادید و می‌خواهید ربات را با اطلاعات قبلی بالا بیاورید:

1. ابتدا ربات را روی سرور جدید نصب کنید (دستورات نصب سریع را اجرا کنید).
2. فایل‌های بکاپ (فایل `.env` و فایل `.sql.gz`) را به سرور جدید و پوشه `/opt/telegramsellbot` منتقل کنید.
3. فایل `.env` بکاپ را جایگزین فایل `.env` فعلی کنید (مطمئن شوید کلیدهای امنیتی تغییر نکنند).
4. کانتینرها را با `docker compose -f docker-compose.prod.yml up -d` دوباره راه‌اندازی کنید.
5. فایل بکاپ دیتابیس را بازیابی کنید:

```bash
cd /opt/telegramsellbot
./restore.sh backups/telegramsellbot_backup_XXXXXX_XXXXXX.sql.gz
```

این دستور دیتابیس جدید را پاک کرده و اطلاعات بکاپ شما را جایگزین می‌کند.

## توسعه محلی

```bash
python -m venv .venv
. .venv/bin/activate
pip install -e .[dev]
pytest -q
```

## ساختار پروژه

- `apps/bot/`: هندلرها، کیبوردها، stateها و middlewareهای ربات
- `apps/api/`: FastAPI، Mini App و وبهوک های پرداخت
- `apps/worker/`: jobهای پس زمینه
- `models/`: مدل های دیتابیس
- `repositories/`: دسترسی به داده و تنظیمات
- `services/`: پرداخت، provisioning، کیف پول، X-UI، نوتیفیکیشن و منطق دامنه
- `miniapp/`: فرانت Mini App
- `tests/`: تست های رگرشن

## نکات امنیتی و عملیاتی

- مخزن public است، اما نرم افزار proprietary است. فایل [LICENSE](LICENSE) را ببینید.
- در production از secretهای قوی برای دیتابیس، Redis، `APP_SECRET_KEY` و `ADMIN_API_KEY` استفاده کنید.
- `NOWPAYMENTS_IPN_SECRET` را حتما تنظیم کنید.
- `.env` را در Git commit نکنید.
- برای درگاه ها، callback دامنه باید با `WEB_BASE_URL` و SSL معتبر بالا باشد.
- پوشه `migrations/` هنوز history کامل Alembic برای همه سناریوهای ارتقا نیست؛ در deploy فعلی bootstrap از metadata هم استفاده می کند.
