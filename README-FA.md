# TelegramSellBot

English README: [README.md](README.md)

TelegramSellBot یک ربات سلف‌هاست برای فروش و مدیریت اشتراک VPN یا Proxy در تلگرام است. این پروژه فقط یک بات ساده نیست؛ API، وبهوک پرداخت، پردازش‌های پس‌زمینه، اتصال به X-UI و اسکریپت‌های استقرار را هم در یک مخزن کنار هم می‌آورد.

## آخرین قابلیت‌های اضافه‌شده

- فروش کانفیگ آماده برای زمانی که اتصال به پنل X-UI ممکن نیست؛ ادمین می‌تواند پلن آماده بسازد، فایل متنی کانفیگ‌ها را آپلود کند و ربات بعد از پرداخت، کانفیگ‌ها را به ترتیب به مشتری تحویل بدهد.
- پنل کاربری Mini App برای فروشگاه، کیف پول، افزایش موجودی، تمدید، سرویس‌های فعال، تیکت، دعوت دوستان و بررسی مجدد پرداخت‌ها بدون انتقال کاربر به ادامه مراحل داخل چت.
- پنل مدیریت Mini App برای آمار، مالی، کاربران، مشتریان، سرویس‌ها، پلن‌ها، موجودی کانفیگ آماده، سرورها، تیکت‌ها، تخفیف‌ها، تنظیمات، لاگ‌ها و اکشن‌های عملیاتی.
- تیکتینگ داخل Mini App برای کاربر و ادمین، شامل نمایش گفتگو، پاسخ ادمین و بستن تیکت.
- بازبینی و بررسی مجدد پرداخت‌های درگاهی برای مواردی که callback با تاخیر می‌رسد.
- رابط کاربری تمیزتر Mini App با آیکون‌های SVG داخلی به جای ناوبری و کارت‌های مبتنی بر ایموجی.

## نصب روی سرور

### نصب یک‌خطی

روی یک سرور Ubuntu و با کاربر `root` این دستور را اجرا کنید:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/sina99778/telegramsellbot/master/setup.sh)
```

این دستور:

- پیش‌نیازهای پایه را نصب می‌کند
- پروژه را داخل `/opt/telegramsellbot` می‌ریزد
- نصاب تعاملی را اجرا می‌کند

### نصب دستی روی سرور

اگر بخواهید همه‌چیز را دستی انجام بدهید:

```bash
sudo -i
apt-get update
apt-get install -y git curl rsync
git clone https://github.com/sina99778/telegramsellbot.git /opt/telegramsellbot
cd /opt/telegramsellbot
chmod +x setup.sh install.sh deploy.sh
bash install.sh
```

بعد از آن، نصاب تعاملی این کارها را برایت جلو می‌برد:

- ساخت `.env`
- نصب Docker، Nginx و Certbot
- استقرار API، bot، worker، PostgreSQL و Redis
- reload سرویس‌ها
- آپدیت فایل‌های پروژه بدون دست‌زدن به `.env`

## این پروژه چه کاری انجام می‌دهد

- فروش پلن از داخل تلگرام
- ساخت خودکار کانفیگ بعد از پرداخت موفق
- فروش کانفیگ آماده از موجودی فایل متنی، بدون نیاز به اتصال X-UI
- پشتیبانی از تمدید، شارژ کیف پول، کد تخفیف و تیکت
- ارائه Mini App برای مسیرهای کاربر و ادمین
- دریافت وبهوک از درگاه‌های پرداخت
- اجرای jobهای زمان‌بندی‌شده برای اعلان انقضا، reconciliation، برودکست، بکاپ و مانیتورینگ

## تکنولوژی‌ها

- بات: `aiogram`
- API: `FastAPI`
- دیتابیس: `PostgreSQL`
- کش و هماهنگی: `Redis`
- ORM: `SQLAlchemy`
- زمان‌بندی: `APScheduler`
- سرویس‌های خارجی: `NOWPayments`، `TetraPay`، `Sanaei X-UI`

## بخش‌های اصلی پروژه

- `apps/bot/` هندلرها، کیبوردها، stateها و middlewareهای ربات
- `apps/api/` اپ FastAPI، مسیرهای ادمین، mini-app و وبهوک‌های پرداخت
- `apps/worker/` jobهای پس‌زمینه
- `services/` منطق پرداخت، کیف پول، provision، نوتیفیکیشن و X-UI
- `models/` و `repositories/` لایه داده، شامل موجودی کانفیگ آماده
- `core/` تنظیمات، bootstrap دیتابیس، امنیت و ابزارهای مشترک
- `tests/` تست‌های رگرشن مسیرهای حساس

## قابلیت‌ها

## آموزش فروش کانفیگ آماده

از فروش کانفیگ آماده وقتی استفاده کنید که نمی‌خواهید یا نمی‌توانید پنل X-UI را به ربات وصل کنید، یا یک لیست آماده از کانفیگ‌ها دارید و می‌خواهید ربات آن‌ها را یکی‌یکی به مشتری‌ها تحویل بدهد.

مراحل استفاده:

1. وارد پنل مدیریت ربات یا پنل مدیریت Mini App شوید.
2. بخش `فروش کانفیگ آماده` را باز کنید.
3. یک پلن آماده بسازید و برای آن نام، مدت، حجم و قیمت تعیین کنید.
4. یک فایل `.txt` آپلود کنید یا متن کانفیگ‌ها را وارد کنید؛ هر خط باید دقیقاً یک کانفیگ کامل باشد.
5. موجودی/Pool آن پلن را فعال نگه دارید.
6. کاربر پلن را مثل پلن‌های عادی از فروشگاه می‌خرد.
7. بعد از تایید پرداخت کیف پول یا درگاه، ربات اولین کانفیگ آزاد از لیست را برمی‌دارد، آن را فروخته‌شده علامت می‌زند، اشتراک را می‌سازد و همان کانفیگ را برای کاربر می‌فرستد.

نکات مهم:

- هر خط فقط به یک مشتری تحویل داده می‌شود.
- خط‌های خالی و کانفیگ‌های تکراری هنگام آپلود نادیده گرفته می‌شوند.
- اگر موجودی کانفیگ آماده تمام شود، آن پلن از لیست خرید کاربر مخفی می‌شود.
- برای هر پلن می‌توانید محدودیت تعداد فروش هم بگذارید. مقدار `0` یعنی فروش نامحدود و محدودیت به کاربر نمایش داده نمی‌شود.
- در پلن‌های آماده، موجودی قابل نمایش بر اساس تعداد کانفیگ‌های آماده موجود هم محدود می‌شود؛ یعنی حتی اگر محدودیت فروش پلن نامحدود باشد، وقتی فایل کانفیگ خالی شود پلن قابل خرید نیست.
- مدت پلن از زمان تحویل کانفیگ حساب می‌شود. تغییر مدت پلن فقط روی خریدهای بعدی اثر دارد و اشتراک‌های قبلی را تغییر نمی‌دهد.
- اگر کاربر پیام کانفیگ را ندید، ادمین می‌تواند از بخش recovery/support کانفیگ تحویل‌داده‌شده را دوباره ارسال کند.

## نکات عملیاتی مهم

- مخزن عمومی است، اما کد همچنان proprietary است. فایل [LICENSE](LICENSE) را ببینید.
- در محیط production باید `NOWPAYMENTS_IPN_SECRET` حتما تنظیم شود.
- برای app، دیتابیس، Redis و دسترسی ادمین از secretهای قوی و یکتا استفاده کنید.
- مسیر فعلی bootstrap دیتابیس هنوز به ساخت schema از روی metadata متکی است.
- پوشه `migrations/` هنوز history کامل Alembic برای همه سناریوهای ارتقا را پوشش نمی‌دهد.
- برای نصب یک‌خطی باید از `setup.sh` استفاده شود، نه `install.sh`، چون `setup.sh` اول مخزن را روی سرور دریافت می‌کند
